HTML Tags Documentation

<base> Tag

The <base> element sets a base URL for all relative URLs in the document. This means that any relative URLs (like those used in hyperlinks and images) will be resolved relative to this base URL. This helps avoid the need to use absolute URLs for every reference in your HTML, simplifying URL management and making it easier to maintain the document.

Attributes:

  • href: Specifies the base URL for all relative URLs in the document. For example, if you set href="https://example.com", then a relative link like <a href="page.html">Link</a> will be resolved to https://example.com/page.html.
  • target: Sets the default target for all hyperlinks and forms. Common values include:
    • _blank: Opens the link in a new tab or window.
    • _self: Opens the link in the same frame (default behavior).
    • _parent: Opens the link in the parent frame.
    • _top: Opens the link in the full window.
  • id, class, style, title: These attributes can be used for additional styling or identification, though they are less commonly applied to <base>.
<base href="https://example.com" target="_blank">

<template> Tag

The <template> element is used to hold client-side content that is not rendered when the page loads. Instead, it provides a way to define reusable HTML snippets that can be used dynamically via JavaScript. This is particularly useful for scenarios where you need to generate content on the fly or repeatedly reuse HTML structures.

Attributes:

  • id: A unique identifier for the template, allowing it to be accessed via JavaScript.
  • class: Assigns a class to the template for styling purposes, though itâ€™s often not used since the template content isn't rendered initially.
  • style: Applies CSS styles directly to the template, though this is less common.
  • title: Provides additional descriptive information, often for accessibility or tooltips.
<template id="my-template">
    <div class="template-content">
        <h2>This is a template</h2>
        <p>This content will not be rendered until it is used programmatically.</p>
    </div>
</template>

<canvas> Tag

The <canvas> element provides a space for dynamic graphics rendering through scripting (usually JavaScript). The canvas API allows you to draw shapes, text, images, and other graphics on the fly. This is ideal for applications requiring real-time graphics, such as games or data visualizations.

Attributes:

  • width: Specifies the width of the canvas in pixels. If not specified, the default width is 300 pixels.
  • height: Specifies the height of the canvas in pixels. The default height is 150 pixels if not specified.
  • id: Assigns a unique identifier to the canvas element for access via JavaScript.
  • class: Allows for CSS styling by applying one or more class names to the canvas.
  • style: Applies inline CSS styles directly to the canvas element.
  • title: Provides additional information about the canvas, often used for tooltips or accessibility.
<canvas id="myCanvas" width="500" height="400"></canvas>
<script>
    var canvas = document.getElementById('myCanvas');
    var ctx = canvas.getContext('2d');
    ctx.fillStyle = 'blue';
    ctx.fillRect(20, 20, 150, 100);
    ctx.strokeStyle = 'red';
    ctx.strokeRect(20, 20, 150, 100);
</script>

<svg> Tag

The <svg> element is used to create scalable vector graphics. Unlike raster images, vector graphics do not lose quality when scaled to different sizes, making SVGs ideal for high-resolution displays and responsive design. SVGs can be created using various shapes, paths, and text elements, and styled or animated with CSS and JavaScript.

Attributes:

  • width: Specifies the width of the SVG viewport. This can be defined in pixels or percentages. For example, width="200" sets the width to 200 pixels.
  • height: Specifies the height of the SVG viewport. Like width, this can be set in pixels or percentages. For instance, height="200" sets the height to 200 pixels.
  • viewBox: Defines the coordinate system and scaling for the SVG. It consists of four values: min-x, min-y, width, and height. This allows the SVG to scale its content proportionally within the defined viewport.
  • id: Provides a unique identifier for the SVG element, which can be used for styling with CSS or manipulation with JavaScript.
  • class: Assigns one or more class names to the SVG, allowing for styling with CSS.
  • style: Applies inline CSS styles directly to the SVG element.
  • title: Offers additional descriptive information about the SVG, often used for tooltips or accessibility purposes.
<svg width="200" height="200" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="yellow" />
    <rect x="10" y="10" width="30" height="30" stroke="black" stroke-width="2" fill="blue" />
    <text x="50" y="90" font-size="12" text-anchor="middle">SVG</text>
</svg>
