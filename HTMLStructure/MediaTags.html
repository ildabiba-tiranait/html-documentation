Media Tags: <img>, <figure>, <figcaption>, <audio>, <video>, <source>, <track>, <object>, <embed>, and <iframe>

<img>

The <img> element is used to embed images into a web document. It allows you to display visual content such as photos, illustrations, or graphics.

The <img> element is self-contained, meaning it does not have a closing tag and it does not contain any content between opening and closing tags.

Attributes:
  • src: Specifies the path to the image file. This attribute is required.
  • alt: Provides alternative text for the image. This is important for accessibility and in case the image cannot be displayed.
  • width: Specifies the width of the image. Can be a number (pixels) or a percentage.
  • height: Specifies the height of the image. Can be a number (pixels) or a percentage.
  • usemap: Specifies an image as a client-side image-map, allowing interactive areas within the image.
  • longdesc: Specifies a URL to a detailed description of the image. Useful for providing additional context.
  • id: Provides a unique identifier for the <img> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <img> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <img> element.
  • title: Provides additional information about the <img> element, usually displayed as a tooltip when the user hovers over the image.
<img src="path/to/image.jpg" alt="Description of image" width="300" height="200" usemap="#map" longdesc="details.html" id="myImage" class="responsive" style="border: 1px solid #ddd;" title="Sample Image">

<figure>

The <figure> element is used to encapsulate self-contained content, such as illustrations, diagrams, photos, or code listings. It is often used in conjunction with the <figcaption> element to provide a caption or description for the content.

The <figure> element allows the content to be referenced and grouped together with a caption, making it easier to maintain the relationship between the content and its description.

Attributes:
  • id: Provides a unique identifier for the <figure> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <figure> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <figure> element.
  • title: Provides additional information about the <figure> element, usually displayed as a tooltip when the user hovers over the figure.
<figure id="myFigure" class="figure" style="text-align: center;" title="Figure Example"> <img src="path/to/image.jpg" alt="Description"> <figcaption>A descriptive caption for the image.</figcaption> </figure>

<figcaption>

The <figcaption> element provides a caption or description for the <figure> element. It is used to describe or provide context for the content contained within the <figure>.

The <figcaption> element should be placed as the first or last child within the <figure> element.

Attributes:
  • id: Provides a unique identifier for the <figcaption> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <figcaption> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <figcaption> element.
  • title: Provides additional information about the <figcaption> element, usually displayed as a tooltip when the user hovers over the caption.
<figure> <img src="path/to/image.jpg" alt="Description"> <figcaption id="caption" class="caption" style="font-style: italic;" title="Caption Example">A detailed caption for the image.</figcaption> </figure>

<audio>

The <audio> element is used to embed sound content, such as music or speech, into a document. It allows users to listen to audio directly from the web page.

The <audio> element can include multiple audio files for different formats, ensuring compatibility with various browsers.

Attributes:
  • src: Specifies the URL of the audio file to be played. It is often used with the <source> element instead.
  • controls: Specifies that audio controls (play, pause, volume) should be displayed.
  • autoplay: Specifies that the audio should start playing as soon as it is ready.
  • loop: Specifies that the audio should start over again when it finishes.
  • muted: Specifies that the audio output should be muted.
  • id: Provides a unique identifier for the <audio> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <audio> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <audio> element.
  • title: Provides additional information about the <audio> element, usually displayed as a tooltip when the user hovers over the audio controls.
<audio src="path/to/audio.mp3" controls autoplay loop muted id="audioPlayer" class="audio" style="width: 100%;" title="Audio Example"></audio>

<video>

The <video> element is used to embed video content in a document. It allows users to watch video clips directly from the web page.

The <video> element can include multiple video files for different formats and provide controls for playback.

Attributes:
  • src: Specifies the URL of the video file. It is often used with the <source> element instead.
  • controls: Specifies that video controls (play, pause, volume) should be displayed.
  • autoplay: Specifies that the video should start playing as soon as it is ready.
  • loop: Specifies that the video should start over again when it finishes.
  • muted: Specifies that the video output should be muted.
  • poster: Specifies an image to be shown while the video is downloading or until the user hits the play button.
  • width: Specifies the width of the video player.
  • height: Specifies the height of the video player.
  • id: Provides a unique identifier for the <video> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <video> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <video> element.
  • title: Provides additional information about the <video> element, usually displayed as a tooltip when the user hovers over the video player.
<video src="path/to/video.mp4" controls autoplay loop muted poster="path/to/poster.jpg" width="640" height="360" id="videoPlayer" class="video" style="border: 1px solid #ddd;" title="Video Example"></video>

<source>

The <source> element is used inside the <audio> and <video> elements to specify multiple media resources. It allows you to provide different formats of the same media file to ensure compatibility with various browsers.

The <source> elements are selected based on their format and the browser's capabilities.

Attributes:
  • src: Specifies the URL of the media file.
  • type: Specifies the media type of the media file, such as audio/mp3 or video/mp4.
  • id: Provides a unique identifier for the <source> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <source> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <source> element.
  • title: Provides additional information about the <source> element, usually displayed as a tooltip when the user hovers over the source.
<video controls> <source src="path/to/video.mp4" type="video/mp4"> <source src="path/to/video.webm" type="video/webm"> Your browser does not support the video tag. </video>

<track>

The <track> element provides text tracks for <video> and <audio> elements. These tracks can be used for subtitles, captions, descriptions, or chapters.

The <track> element should be used inside the <video> or <audio> element to provide supplementary text content.

Attributes:
  • src: Specifies the URL of the track file.
  • kind: Specifies the kind of text track. Values include:
    • subtitles: Subtitles for the video.
    • captions: Captions for the video.
    • descriptions: Descriptions of the video content.
    • chapters: Chapters or sections of the video.
  • srclang: Specifies the language of the track text data.
  • label: Specifies a user-readable title for the text track.
  • default: Specifies that the track should be enabled if the user's preferences do not indicate that another track would be more appropriate.
  • id: Provides a unique identifier for the <track> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <track> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <track> element.
  • title: Provides additional information about the <track> element, usually displayed as a tooltip when the user hovers over the track.
<video controls> <source src="path/to/video.mp4" type="video/mp4"> <track src="path/to/subtitles_en.vtt" kind="subtitles" srclang="en" label="English"> <track src="path/to/subtitles_es.vtt" kind="subtitles" srclang="es" label="Spanish"> Your browser does not support the video tag. </video>

<object>

The <object> element embeds an external resource, such as a plugin or multimedia content, into the document. It is used for displaying objects like Flash animations or PDFs.

The <object> element can specify a fallback content to display if the object cannot be loaded.

Attributes:
  • data: Specifies the URL of the resource to be used by the object.
  • type: Specifies the media type of the resource, such as application/pdf or image/jpeg.
  • width: Specifies the width of the object.
  • height: Specifies the height of the object.
  • usemap: Specifies an image as a client-side image-map.
  • name: Specifies the name of the object, used for identifying the object.
  • id: Provides a unique identifier for the <object> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <object> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <object> element.
  • title: Provides additional information about the <object> element, usually displayed as a tooltip when the user hovers over the object.
<object data="path/to/resource.pdf" type="application/pdf" width="600" height="400" id="pdfViewer" class="viewer" style="border: 1px solid #ddd;" title="PDF Viewer"> Your browser does not support PDFs. Download the PDF. </object>

<embed>

The <embed> element embeds external content, typically multimedia plugins, into the document. It is used for content such as interactive maps or other applications.

Unlike the <object> element, the <embed> element is often used for self-contained content and does not require additional fallback content.

Attributes:
  • src: Specifies the URL of the resource to be embedded.
  • type: Specifies the media type of the embedded content, such as application/x-shockwave-flash.
  • width: Specifies the width of the embedded content.
  • height: Specifies the height of the embedded content.
  • id: Provides a unique identifier for the <embed> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <embed> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <embed> element.
  • title: Provides additional information about the <embed> element, usually displayed as a tooltip when the user hovers over the embedded content.
<embed src="path/to/plugin.swf" type="application/x-shockwave-flash" width="800" height="600" id="flashPlugin" class="plugin" style="border: 1px solid #ddd;" title="Flash Plugin">

<iframe>

The <iframe> element embeds another HTML page within the current document. It allows you to display external content, such as another website, within a frame on your page.

The <iframe> element can be used for a variety of purposes, including embedding maps, videos, and external applications.

Attributes:
  • src: Specifies the URL of the document to embed.
  • width: Specifies the width of the iframe.
  • height: Specifies the height of the iframe.
  • name: Specifies the name of the iframe, which can be used as a target for hyperlinks.
  • sandbox: Enables an extra set of restrictions for the content in the iframe, such as preventing form submissions or script execution.
  • allow: Specifies a feature policy for the iframe, such as allowing access to the camera or microphone.
  • id: Provides a unique identifier for the <iframe> element, useful for targeting with CSS or JavaScript.
  • class: Assigns one or more class names to the <iframe> element for styling with CSS.
  • style: Applies inline CSS styles directly to the <iframe> element.
  • title: Provides additional information about the <iframe> element, usually displayed as a tooltip when the user hovers over the iframe.
<iframe src="https://example.com" width="800" height="600" name="myIframe" sandbox="allow-same-origin allow-scripts" allow="camera; microphone" id="myIframe" class="iframe" style="border: 1px solid #ddd;" title="Embedded Content"></iframe>
