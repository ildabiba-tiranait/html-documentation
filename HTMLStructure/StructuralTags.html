HTML Structural Tags Documentation

<html> Tag

The <html> element is the root element of an HTML document. It encompasses all other HTML elements (except for the <!DOCTYPE> declaration). This tag provides the structure for the HTML document and is essential for defining the document's language using the lang attribute.

Attributes:

  • lang: Specifies the language of the document's content. For example, lang="en" for English or lang="fr" for French.
<html lang="en">
    <head>...</head>
    <body>...</body>
</html>

<head> Tag

The <head> element contains metadata and links to external resources for the HTML document. It includes elements like <title>, <meta>, <link>, <style>, and <script>. This section does not display content directly but provides essential information and resources for the document.

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
    <link rel="stylesheet" href="styles.css">
    <script src="script.js"></script>
</head>

<body> Tag

The <body> element encloses all the content of the HTML document, including text, images, links, and other media. It is the main content area where users interact with the page. This tag previously included attributes like background, bgcolor, text, link, vlink, and alink, but these attributes are deprecated in modern HTML.

Attributes: (Deprecated)

  • background: Specifies a background image for the document. Deprecated.
  • bgcolor: Sets the background color. Deprecated.
  • text: Sets the color of the text. Deprecated.
  • link: Sets the color of unvisited links. Deprecated.
  • vlink: Sets the color of visited links. Deprecated.
  • alink: Sets the color of active links. Deprecated.
<body>
    <h1>Hello, World!</h1>
    <p>Welcome to my website.</p>
</body>

<header> Tag

The <header> element represents introductory content for a section or the whole document. It often includes headings, navigation links, and other introductory elements. It is used to group together elements that provide context or navigation for the page or section.

Attributes:

  • id: Provides a unique identifier for the header element.
  • class: Assigns one or more class names for styling purposes.
  • style: Applies inline CSS styles directly to the header element.
  • title: Offers additional information about the header element, often displayed as a tooltip.
<header>
    <h1>Site Title</h1>
    <nav>
        <a href="#home">Home</a>
        <a href="#about">About</a>
    </nav>
</header>

<footer> Tag

The <footer> element represents the footer of a document or section. It is typically used to contain contact information, copyright notices, or related links. It provides additional context or resources at the end of a document or section.

<main> Tag

The <main> element indicates the primary content of a document, excluding headers, footers, and sidebars. It helps search engines and assistive technologies understand the main content of the page. There should be only one <main> element per document.

Attributes:

  • id: Provides a unique identifier for the main element.
  • class: Assigns one or more class names for styling purposes.
  • style: Applies inline CSS styles directly to the main element.
  • title: Offers additional information about the main element, often displayed as a tooltip.
<main>
    <h2>Main Content</h2>
    <p>This is the primary content of the page.</p>
</main>

<section> Tag

The <section> element defines a section within a document. It is typically used to group related content together, often with its own heading. Sections can be used to divide content into thematic areas.

Attributes:

  • id: Provides a unique identifier for the section element.
  • class: Assigns one or more class names for styling purposes.
  • style: Applies inline CSS styles directly to the section element.
  • title: Offers additional information about the section element, often displayed as a tooltip.
<section>
    <h2>Section Title</h2>
    <p>Content for this section goes here.</p>
</section>

<article> Tag

The <article> element represents a self-contained piece of content, such as a blog post, news article, or user comment. Articles are meant to be independently distributable or reusable.

Attributes:

  • id: Provides a unique identifier for the article element.
  • class: Assigns one or more class names for styling purposes.
  • style: Applies inline CSS styles directly to the article element.
  • title: Offers additional information about the article element, often displayed as a tooltip.
<article>
    <h2>Article Title</h2>
    <p>This is a self-contained piece of content.</p>
</article>

<nav> Tag

The <nav> element defines a set of navigation links. It helps users and search engines identify the sections of the page dedicated to navigation. Typically, it includes a list of links to other sections or pages.

Attributes:

  • id: Provides a unique identifier for the nav element.
  • class: Assigns one or more class names for styling purposes.
  • style: Applies inline CSS styles directly to the nav element.
  • title: Offers additional information about the nav element, often displayed as a tooltip.
<nav>
    <a href="#home">Home</a>
    <a href="#services">Services</a>
    <a href="#contact">Contact</a>
</nav>

<aside> Tag

The <aside> element contains content tangentially related to the main content, such as sidebars, pull quotes, or additional information. It provides supplementary content that adds context or extra information to the main content.

Attributes:

  • id: Provides a unique identifier for the aside element.
  • class: Assigns one or more class names for styling purposes.
  • style: Applies inline CSS styles directly to the aside element.
  • title: Offers additional information about the aside element, often displayed as a tooltip.
<aside>
    <p>This is supplementary content related to the main article.</p>
</aside>

<div> Tag

The <div> element is a generic container used to group and style content. It does not affect the layout by itself but can be used to apply CSS styles or JavaScript functions to a block of content. It is often used for structuring and organizing elements.

Attributes:

  • id: Provides a unique identifier for the div element.
  • class: Assigns one or more class names for styling purposes.
  • style: Applies inline CSS styles directly to the div element.
  • title: Offers additional information about the div element, often displayed as a tooltip.
<div>
    <p>This is a generic container for content.</p>
</div>
