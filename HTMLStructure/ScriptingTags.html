HTML Tags Documentation

<script> Tag

The <script> element is used to define client-side scripts, usually written in JavaScript. This tag can either include JavaScript code directly or reference an external JavaScript file. It can be placed in the document's head or body, depending on when you want the script to be executed.

Attributes:

  • src: Specifies the URL of an external script file. If this attribute is provided, the script content is loaded from the given URL. For example, src="script.js".
  • type: Specifies the scripting language. The default is text/javascript. In modern HTML5, this attribute is optional as JavaScript is the default scripting language.
  • async: Indicates that the script should be executed asynchronously as soon as it is available. This is useful for scripts that do not depend on the DOM or other scripts. However, scripts with this attribute may execute in any order.
  • defer: Indicates that the script should be executed after the page has finished parsing. This ensures that the script runs in the order it appears in the document, but only after the HTML is completely loaded.
  • crossorigin: Handles cross-origin requests for the script. Values include:
    • anonymous: Requests are made without credentials.
    • use-credentials: Requests are made with credentials, such as cookies or HTTP authentication.
  • integrity: Provides a way to verify that the fetched script has not been altered by comparing it to a specified cryptographic hash. This helps ensure the script's integrity and security.
  • id: Assigns a unique identifier to the script element, which can be used for styling or scripting purposes.
  • class: Allows for CSS styling by applying one or more class names to the script element.
  • style: Applies inline CSS styles directly to the script element, though itâ€™s not commonly used.
  • title: Provides additional descriptive information about the script element, often used for tooltips or accessibility.

<noscript> Tag

The <noscript> element is used to define alternate content for users whose browsers do not support client-side scripting or have JavaScript disabled. This tag ensures that users without JavaScript can still receive meaningful content or instructions.

Attributes:

  • id: Provides a unique identifier for the <noscript> element, which can be used for styling or scripting purposes.
  • class: Assigns one or more class names to the <noscript> element, allowing for CSS styling.
  • style: Applies inline CSS styles directly to the <noscript> element, though this is rarely needed.
  • title: Offers additional descriptive information about the <noscript> element, which can be useful for tooltips or accessibility.
<noscript>
    <p>JavaScript is required to view this content. Please enable JavaScript in your browser settings.</p>
</noscript>
